项目简介
    **智规引擎** 是一款面向**国家标准/法规 PDF 文档**的智能化知识库问答系统。区别于通用 RAG 方案，本项目针对标准文档"章-节-条-款"的强层级结构进行了深度优化
      ，通过**标题感知分块、知识图谱增强、相邻块召回、多 Agent 质量审核**等机制，解决传统 RAG 在结构化长文档中面临的检索不全、上下文断裂、引用缺失等核心问题。

后期会继续完善这个项目：
  完善方向：
    针对查询不同的查询类型：
      用LLM判断查询意图，路由到不同的检索 pipeline。
      示例：
        | 查询                                | 路由到      | 原因     |
        | ---------------------------------   | -------    | ------ |
        | 建筑设计防火规范的主要内容是什么？                  | summary | 概述类问题  |
        | 耐火等级二级的多层建筑疏散距离是多少？              | detail  | 具体参数查询 |
        | GB 50016 和 GB 50045 的防火分区要求有什么不同？    | compare | 多文档比较  |
        | 帮我总结一下第三章的内容                           | summary | 章节摘要   |
    多agent设计：
      Orchestrator Agent（编排器，总调度）
      │
      ├── Retriever Agent（检索）
      │   - 向量检索 / BM25 / 图谱检索
      │
      ├── Analyzer Agent（分析）
      │   - 比较 / 验证 / 总结
      │
      └── Generator Agent（生成）
          - 答案组装 / 引用标注 / 格式输出
      示例：
        1. Orchestrator Agent
           分析意图：compare（多文档比较）
           制定计划：
             - Step 1: 从 GB50016 检索"高层建筑 疏散楼梯"
             - Step 2: 从 GB50045 检索"高层建筑 疏散楼梯"
             - Step 3: 比较两组检索结果
             - Step 4: 生成最终答案
        
        2. Retriever Agent A（并行）
           查询：GB50016 + 高层建筑 + 疏散楼梯
           检索策略：混合检索（向量 + BM25）→ Cross-Encoder 重排序
           返回：5 个最相关的 chunk（来自 GB50016 第 6 章）
        
        3. Retriever Agent B（并行）
           查询：GB50045 + 高层建筑 + 疏散楼梯
           检索策略：同上
           返回：5 个最相关的 chunk（来自 GB50045 第 6 章）
        
        4. Comparator Agent
           输入：Retriever A 和 B 的结果
           分析：提取两个标准中的关键参数（疏散宽度、楼梯间形式、前室要求等）
           输出：对比表格 + 差异总结
        
        5. Generator Agent
           输入：Comparator 的分析结果 + 原始 chunk
           组装上下文：按对比维度组织引用
           生成答案：两个标准对高层建筑疏散楼梯的要求主要区别如下：...
        
        6. Orchestrator Agent（汇总）
           整合各 Agent 输出
           添加引用标注
           返回给用户
